# CI/CD

## 운영 방식

- 이 문서는 CI/CD 학습 인덱스로만 사용한다.
- 상세 내용은 `200-practical-backend/ci-cd/` 아래 개념별 문서로 나눈다.
- 하루에 학습한 내용만 하나의 작은 문서에 정리한다.
- 아직 학습하지 않은 내용을 미리 길게 채우지 않는다.
- 각 문서는 빌드, 테스트, 배포, 롤백, 운영 안정성, 면접 답변을 중심으로 작성한다.

## 학습 순서

1. CI와 CD의 차이
2. Build Pipeline
3. Test Automation
4. Deployment Strategy
5. Rollback
6. Environment Variable
7. Secret 관리
8. GitOps

## 핵심 질문

- CI/CD 영역에서 실무적으로 중요한 문제는 무엇인가?
- 문제를 발견하면 어떤 순서로 원인을 좁히는가?
- 어떤 해결책이 있고 각각의 한계는 무엇인가?
- 프로젝트 경험과 어떻게 연결할 수 있는가?
- 면접에서는 어떻게 설명할 수 있는가?

## 실무 관점

- CI/CD는 빠른 배포보다 반복 가능하고 되돌릴 수 있는 배포가 중요하다.
- 배포 자동화는 테스트, 설정, 권한, 롤백 전략과 함께 설계해야 한다.
- 면접에서는 파이프라인 단계와 실패 시 대응 방식을 설명한다.

# Learning Principles

## 원칙

- 매일 새 문서를 많이 만들지 않는다.
- 이미 있는 문서를 계속 보강한다.
- 하나의 개념은 반드시 실무 문제와 연결한다.
- 설명할 수 없는 내용은 아직 아는 것이 아니다.
- AI는 초안 생성과 검증 도구로 사용한다.
- 공식 문서, 실제 코드, 장애 사례, 실행 결과로 검증한다.

## 최종 목표

- 실무에서 문제를 예방하고 원인을 좁힐 수 있다.
- 기술 면접에서 개념, 원리, 장단점, 실무 사례를 연결해 설명할 수 있다.
- 암기한 답변이 아니라 실제 판단 기준을 말할 수 있다.
- 각 문서는 나중에 면접 답변과 실무 체크리스트로 재사용할 수 있어야 한다.

## 매일 할 일

1. 오늘 볼 주제 하나를 고른다.
2. 해당 문서를 연다.
3. 한 줄 정의를 보강한다.
4. 동작 원리나 실무 문제를 추가한다.
5. 면접 질문 하나를 만든다.
6. 관련 문서를 링크한다.
7. 커밋한다.

## 개념 문서에 반드시 남길 것

- 한 줄 정의
- 왜 실무에서 중요한지
- 핵심 동작 원리
- 자주 발생하는 버그나 장애 양상
- 실무 판단 기준
- 면접 답변 1분 버전
- 꼬리 질문

## 커밋 예시

```bash
git push origin main
```

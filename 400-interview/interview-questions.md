# Interview Questions

## 운영 방식

- 질문을 외우지 않는다.
- 질문마다 핵심 답변, 실무 예시, 주의점을 함께 정리한다.
- 프로젝트 경험과 연결할 수 있는 질문은 반드시 연결한다.

## Java / Spring / JPA

- Java 버전별 차이는 무엇인가?
- Spring을 왜 사용하나요?
- DI는 왜 필요한가요?
- @Transactional은 어떻게 동작하나요?
- JPA를 왜 사용하나요?
- N+1은 왜 발생하나요?

## DB / Redis / Concurrency

- 실행 계획에서 무엇을 확인하나요?
- 복합 인덱스 컬럼 순서는 어떻게 정하나요?
- Redis 캐싱 적용 시 정합성 문제를 어떻게 다루나요?
- 멱등성은 어떻게 보장하나요?

## Infra / Observability

- Kubernetes CronJob을 왜 사용하나요?
- GitOps의 장점은 무엇인가요?
- 장애가 발생하면 어떤 순서로 원인을 좁히나요?
- traceId는 왜 필요한가요?
